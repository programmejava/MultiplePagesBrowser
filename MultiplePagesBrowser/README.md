# MultiGrid Browser

> 基于 **WPF + WebView2 (.NET 10)** 构建的多格子浏览器，可同时在一个窗口内显示 1 / 4 / 9 / 12 / 16 个独立的网页格子，适合多页面监控、多账号操作、信息对比等场景。

## 功能概览

| 功能 | 说明 |
|---|---|
| 多格子布局 | 支持 1 / 4 / 9 / 12 / 16 格，随时切换，不销毁已有页面 |
| 独立格子 | 每个格子有独立的 WebView2 实例，互不干扰 |
| Cookie 共享/隔离 | 一键切换：所有格子共享登录状态，或每个格子独立账号 |
| 视频自动暂停 | 切换激活格子时，其他格子的视频自动暂停，切回时恢复 |
| 内存降级优化 | 非激活格子通知 Chromium 回收渲染缓存，大幅降低内存占用 |
| 防页面劫持 | 拦截 `target="_blank"` 等新窗口请求，在当前格子内加载 |
| 粘贴导航 | 在格子内按 Ctrl+V，若剪贴板是 URL 则直接导航 |
| 复制链接 | 在格子内按 Ctrl+C（无选中文字时）复制当前页面链接 |
| 右键菜单扩展 | 粘贴并打开链接 / 复制页面链接 / 用外部浏览器打开 |
| 同步导航 | 将同一网址同时推送给所有格子 |
| 全局静音 | 一键静音所有格子 |
| 设置持久化 | 所有设置保存到本地 JSON，重启后自动恢复 |

## 快速上手

1. 编译并运行项目（Visual Studio 2022+ 或 `dotnet run`）
2. 程序启动后默认显示 **9 格（3×3）** 布局
3. 点击任意格子将其**激活**（蓝色边框高亮）
4. 在顶部地址栏输入 URL，按 **Enter** 在激活格子中打开
5. 点击工具栏布局按钮（1格 / 4格 / 9格 / 12格 / 16格）切换布局

## 布局切换

点击工具栏的布局按钮，或按以下规则理解布局：

| 按钮 | 格子数 | 排列 |
|---|---|---|
| 1格 | 1 | 1×1 |
| 4格 | 4 | 2×2 |
| 9格 | 9 | 3×3（默认）|
| 12格 | 12 | 4×3 |
| 16格 | 16 | 4×4 |

**切换布局时**：
- 已存在的格子（WebView2 实例）被**复用**，不会重新加载页面
- 新增的格子加载**主页**（若主页未设置则显示空白页）
- 格子数减少时，多余格子仅从视图隐藏，页面状态保留

## 导航操作

### 导航当前格子

1. 点击目标格子激活它（蓝色边框）
2. 在地址栏输入 URL
3. 按 **Enter**

### 同步导航所有格子

- 方式一：在地址栏输入 URL 后按 **Ctrl + Enter**
- 方式二：输入 URL 后点击工具栏「同步导航」按钮

### 主页按钮

- 点击 🏠 按钮，激活格子跳转到在**设置**中配置的主页
- 若主页为空，则显示空白页

### 在格子内导航

- 点击页面内的链接正常跳转（不会弹出新窗口，均在当前格子内打开）
- 使用工具栏的 ◀ ▶ 按钮在当前格子的历史记录中前进/后退

## 格子操作（快捷键 & 右键菜单）

### 激活格子

- **鼠标点击**页面任意位置即可激活该格子（通过注入 JS `mousedown` 事件实现）

### 快捷键

| 快捷键 | 作用范围 | 说明 |
|---|---|---|
| `Ctrl + C` | 格子内，无选中文字 | 复制当前格子的页面 URL 到剪贴板 |
| `Ctrl + C` | 格子内，有选中文字 | 正常复制选中的文字（不拦截）|
| `Ctrl + V` | 格子内 / 工具栏 | 若剪贴板内容是有效的 http/https URL，在当前激活格子打开 |
| `Ctrl + V` | 地址栏 | 正常粘贴文字（不拦截）|

### 右键菜单

在任意格子内右键，菜单底部会追加三个自定义项：

| 菜单项 | 说明 | 可用条件 |
|---|---|---|
| 📋 粘贴并打开链接 | 将剪贴板中的 URL 在当前格子打开 | 剪贴板含有效 http/https 链接 |
| 🔗 复制页面链接 | 将当前格子的 URL 写入剪贴板 | 格子已有页面（非空白页）|
| 🌐 用外部浏览器打开 | 调用系统默认浏览器打开当前页面 | 格子已有页面（非空白页）|

## Cookie / 登录隔离

程序提供两种 Cookie 模式，通过工具栏的 **🍪 共享登录** 切换按钮控制：

### 共享模式（默认，按钮高亮）

```
所有格子 → 同一个 WebView2 用户数据目录
路径：%LocalAppData%\MultiplePagesBrowser\SharedProfile
```

- 所有格子共享 Cookie、登录状态、缓存
- 在一个格子登录后，其他格子自动识别同一账号
- 适合：多窗口查看同一账号的不同页面

### 独立模式（按钮灰色）

```
格子 1 → %LocalAppData%\MultiplePagesBrowser\Tile_0
格子 2 → %LocalAppData%\MultiplePagesBrowser\Tile_1
...
```

- 每个格子有独立的 Cookie 存储
- 可以在不同格子同时登录同一网站的不同账号
- 适合：多账号运营、账号对比
- ⚠️ 切换模式后需**重新初始化格子**（关闭后重新打开该格子，或重启程序）才能完全生效

## 设置面板

点击工具栏右侧的 **⚙** 按钮，或按 **Ctrl + ,** 打开设置面板。

设置保存路径：`%LocalAppData%\MultiplePagesBrowser\settings.json`

### 通用

| 设置项 | 说明 | 默认值 |
|---|---|---|
| 主页地址 | 打开新格子、点击🏠按钮时加载的页面；留空则为空白页 | 空 |
| 默认布局 | 程序启动时默认使用的格子数 | 9格（3×3）|

### 隐私 / Cookie

| 设置项 | 说明 | 默认值 |
|---|---|---|
| 共享 Cookie / 登录状态 | 等同于工具栏「共享登录」开关 | 开启 |

### 性能优化

| 设置项 | 说明 | 默认值 | 推荐 |
|---|---|---|---|
| 非激活格子暂停视频 | 切换激活格子时，向其他格子注入 JS 暂停 `<video>` 播放，切回时自动恢复 | 开启 | ✅ 推荐开启 |
| 非激活格子降低内存优先级 | 通知 Chromium 对后台格子回收渲染缓存 | 开启 | ✅ 推荐开启 |

### 外观

| 设置项 | 说明 | 默认值 |
|---|---|---|
| 格子间距 (px) | 格子之间的间隔像素数，范围 1~12 | 3 |
| 显示格子编号角标 | 格子左上角显示编号（1、2、3…）| 开启 |

## 性能优化原理

### 16个格子同时运行时的挑战

未优化状态下，16个格子同时播放 YouTube 4K 视频会面临：
- GPU 同时解码 16 路 4K 视频流 → **GPU 使用率 100%，严重掉帧**
- 每个 WebView2 实例约占 800 MB~1 GB 渲染缓存 → **总内存 12~16 GB+**

### 优化策略

#### 策略一：共享浏览器进程

所有格子共享同一个 WebView2 环境（`CoreWebView2Environment`），使用同一个 Chromium 主进程，避免启动 16 个独立的 `msedgewebview2.exe`。

#### 策略二：格子复用池（Tile Pool）

切换布局时，`WebTile` 实例被缓存在 `_tilePool` 字典中，不销毁、不重建 WebView2，切换回来时页面状态完整保留。

#### 策略三：JS 视频暂停（最有效）

```
激活格子 A → 切换到格子 B
↓
向格子 A 注入 JS：
  document.querySelectorAll('video').forEach(v => {
    if(!v.paused){ v.pause(); v.dataset.autoPaused='1'; }
  });

切换回格子 A 时：
  document.querySelectorAll('video').forEach(v => {
    if(v.dataset.autoPaused){ v.play(); delete v.dataset.autoPaused; }
  });
```

效果：同一时刻只有**激活格子**的视频在解码，GPU 解码负载降低约 **90%**。

#### 策略四：Chromium 内存降级

```csharp
// 非激活格子
_webView.CoreWebView2.MemoryUsageTargetLevel =
    CoreWebView2MemoryUsageTargetLevel.Low;

// 激活格子恢复
_webView.CoreWebView2.MemoryUsageTargetLevel =
    CoreWebView2MemoryUsageTargetLevel.Normal;
```

通知 Chromium 对后台进程主动回收 JavaScript 堆、渲染缓存等，可节省约 **30~50%** 内存。

#### 策略五：进程挂起

```csharp
await _webView.CoreWebView2.TrySuspendAsync();  // 非激活时
_webView.CoreWebView2.Resume();                  // 激活时
```

挂起后台格子的 Chromium 渲染进程，几乎消除后台 CPU 占用。

### 优化效果对比

| 场景 | 内存占用 | GPU 占用 |
|---|---|---|
| 16格，全部播放 4K，无优化 | ~16 GB+ | ~100% |
| 16格，全部播放 4K，开启优化 | ~4~6 GB | 仅激活格子解码 |

> **推荐硬件**（16格 4K 视频场景）：32 GB RAM + RTX 3060 及以上

## 常见问题

**Q：切换 Cookie 模式后，登录状态没有变化？**

A：Cookie 模式（共享/独立）在 WebView2 初始化时确定，已打开的格子不会实时切换。切换后请重启程序，或手动刷新所有格子以重新初始化。

---

**Q：格子内的视频不会自动恢复播放？**

A：自动恢复依赖注入的 JS 标记（`data-autoPaused`），部分网站的播放器（如 YouTube）会检测到 `pause()` 调用并更新 UI 状态，恢复时 `play()` 通常有效。若遇到不恢复的情况，手动点击播放即可。

---

**Q：Ctrl+V 粘贴后没有导航？**

A：程序只拦截完整的 `http://` 或 `https://` URL，不含协议前缀的地址（如 `google.com`）不会触发粘贴导航。请在 URL 前加上 `https://`。

---

**Q：用外部浏览器打开没有反应？**

A：程序调用 `Process.Start` 以 `UseShellExecute = true` 方式打开 URL，依赖系统默认浏览器配置。请确保 Windows 中已设置默认浏览器。

---

**Q：16格布局下内存依然很高？**

A：确认设置面板中「非激活格子暂停视频」和「非激活格子降低内存优先级」均已**开启**。另外，各格子加载的网站本身也有内存占用，YouTube 等富媒体网站约 300~500 MB/格。

---

**Q：WebView2 运行时提示未安装？**

A：下载并安装 [Microsoft Edge WebView2 Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)，选择「常青版独立安装程序」即可。
